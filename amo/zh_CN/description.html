访问以下链接可获取 Keywi 更新信息： https://github.com/LEDfan/keywi/blob/master/amo/DESCRIPTION-EN.html 目的
  • Firefox 极简密码集成
  • 在登录输入框填充用户名与密码，支持上下文菜单与快捷键
  • 利用 [Keepass HTTP](https://github.com/pfn/keepasshttp)
  • 不编辑 Keepass database
  • 使用 Firefox web extensions API，支持多进程
  • 支持 Keepass，后续可能支持 [Pass](https://www.passwordstore.org/)
  • 使用最少的库 (aes, cryptoheplers and reqwest)
  • 不支持自动表单填入，也即是仅在用户需要时填充。这点非常重要，因为：
    • 凭证可能在注册表单中填入
    • 凭证可能在隐藏的表单中填入
    • 你也许现在不希望登录
    • 页面包含多个表单（不太可能）
  • 不会在表单中注入按钮或者图标
使用 Keywi 浏览器的本地存储机制以储存连接到 Keepass 需要的信息，包含与 Keepass 交互需要的验证加密密钥。存储在浏览器本地存储的信息未被加密。因此 Keywi 使用基于用户提供的密码的AES-256密钥加密信息 (database.key, database.id and database.hash) 。 密码会在初次运行时询问，Firefox 每次启动时均会问询是否解密信息。 Keywi 需要 Firefox 51 或更高版本。 初次运行
  1. 安装 KeepassHTTP。参见 https://github.com/pfn/keepasshttp
  2. 确认对应的 Keepass database 已打开
  3. 安装扩展
  4. 你会被询问一个用于 Secure Storage 的密码。你 **不应该** 使用与 Keepass 相同的密码，并应当确保该密码足够强。 截图
  5. Keepass 会询问并确认关联。 截图
Firefox 启动时 你会被询问是否解锁 Secure Storage。 Screenshot 凭据填充
  1. 选项用户名填入框
  2. 右击并在“Keywi”菜单中选择“填入用户名与密码”。也可以单独填入用户名或密码。 截图
  3. 当 Keepass 显示获取凭据的许可时，点击“允许”
  4. 完成填入
键盘快捷键 支持以下键盘快捷键：
  • Ctrl + Shift + F 在当前输入框中填入用户名与密码（当未找到密码框时仅填入用户名）
  • Ctrl + Shift + D 仅在当前输入框中添入密码
